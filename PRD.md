# [PRD] 이미지-PDF 변환 및 자동 삭제 웹 애플리케이션 (AntG 배포용)

## 1. 개요 (Overview)

* **제품명:** AntG Image to PDF Converter (가칭)
* **목적:** 사용자가 다중 이미지를 드래그앤드롭 또는 파일 탐색기로 업로드하면, 이를 순서대로 병합하여 하나의 PDF 파일로 생성 및 다운로드할 수 있는 SvelteKit 기반 풀스택 웹 애플리케이션 구축
* **주요 목표:** * **모바일 퍼스트 UX:** 데스크톱과 모바일 터치 환경 모두에서 매끄러운 이미지 관리 체계 제공
* **서버 자원 최적화:** 24시간 후 업로드 및 생성된 모든 파일을 완벽히 자동 제거(TTL)하는 스케줄러 구현
* **안정적인 아키텍처:** 클라이언트 튕김을 방지하기 위해 서버 사이드(SvelteKit API)에서 PDF 병합 프로세스 처리



---

## 2. 사용자 흐름 (User Flow)

1. **진입:** 사용자가 웹 애플리케이션 메인 페이지에 접속한다.
2. **업로드:** 드래그앤드롭 Zone 또는 모바일 터치 업로드를 통해 파일(PNG, JPG, JPEG)을 다중 업로드한다.
3. **검토:** 업로드된 이미지들의 썸네일 그리드가 화면에 정렬된다.
4. **변환 요청:** 'PDF 만들기' 버튼을 누른다.
5. **대기 (로딩):** 화면이 로딩 상태로 전환되며, 세련된 CSS/SVG 스피너/프로그레스 바 애니메이션이 실행된다. (이때 UI 및 버튼은 비활성화된다.)
6. **완료 및 다운로드:** 변환이 완료되면 로딩 애니메이션이 종료되고, 'PDF 다운로드' 버튼이 나타난다. 버튼을 클릭하면 파일이 다운로드된다.

---

## 3. 상세 기능 요구사항 (Functional Requirements)

### 3.1 프론트엔드 (Client-Side)

#### F-01: 반응형 이미지 업로드 & 드래그앤드롭

* **상세 요구사항:**
* 데스크톱 환경에서의 표준 `dragover`, `drop` 이벤트를 지원하는 드래그 영역 UI를 제공해야 함.
* 모바일 환경에서는 터치 한 번으로 시스템 파일 탐색기/앨범이 열리도록 투명 파일 인풋(`input type="file" multiple accept="image/*"`)을 상위에 래핑해야 함.
* 허용 파일 확장자는 `PNG`, `JPG`, `JPEG`로 제한하며, 기타 파일 업로드 시 예외 처리 경고(Toast 알림 등)를 노출함.



#### F-02: 썸네일 그리드 리스트 및 상태 제어

* **상세 요구사항:**
* 업로드된 이미지는 화면 크기에 맞춰 반응형(예: 모바일 2열, 데스크톱 4열 이상) 그리드로 썸네일을 표시함.
* **[우선순위 P1]** 사용자가 모바일 터치 및 데스크톱 드래그를 통해 썸네일의 정렬 순서를 변경할 수 있는 UI 컴포넌트 구조(Sortable 라이브러리 연동 등)를 설계함.



#### F-03: 변환 상태 애니메이션 제어

* **상세 요구사항:**
* 'PDF 만들기' 클릭 시 클라이언트 단에서 이미지 파일을 폼 데이터(`FormData`)에 담아 SvelteKit API 엔드포인트로 전송함.
* 응답이 오기 전까지 `isProcessing = true` 상태를 유지하며, 이 상태 동안에는 전체 업로드 영역 및 버튼이 `disabled` 처리되고 CSS/SVG 기반의 중앙 정렬 로딩 스피너가 활성화됨.
* 성공 응답 수신 시 완료 상태로 전환되며 결과 파일의 다운로드 URL을 버튼에 바인딩함.



---

### 3.2 백엔드 (Server-Side)

#### B-01: API 엔드포인트 및 PDF 병합 프로세스 (`src/routes/api/pdf/+server.js`)

* **상세 요구사항:**
* 클라이언트로부터 전송받은 멀티파트 폼 데이터(`multipart/form-data`)에서 이미지 스트림을 추출함.
* 모바일 기기의 메모리 과부하를 방지하기 위해 무거운 PDF 변환 연산은 서버에서 진행함. (`pdf-lib` 또는 `jsPDF` 라이브러리를 Node.js 백엔드단에서 활용)
* 추출한 이미지를 순서대로 A4 또는 이미지 원본 비율에 맞춰 단일 PDF 객체에 렌더링한 후, 서버의 지정된 임시 디렉토리(`/static/uploads/` 또는 별도 임시 폴더)에 파일로 저장하거나 고유 ID가 부여된 다운로드 링크를 반환함.



#### B-02: 24시간 자동 삭제 스케줄러 (Storage TTL)

* **상세 요구사항:**
* 업로드된 이미지 원본 및 생성된 결과물 PDF 파일은 개인정보 보호 및 서버 용량 확보를 위해 **저장 시점 기준 24시간이 지나면 자동으로 서버에서 영구 삭제**되어야 함.
* **구현 모델 가이드:**
1. `node-cron` 등을 활용해 매 정시(또는 주기적)마다 지정 폴더 내 파일들의 수정 시간(`fs.stat`의 `mtime`)을 체크하여 현시점 기준 24시간 전 파일들을 `fs.unlink`하는 백그라운드 태스크 구현.
2. 서버리스 배포 환경(Vercel, Cloudflare 등)인 경우를 대비하여 프롬프트 레벨에서 외부 클라우드 오리진 스토리지(Vercel Blob, Cloudflare R2 등)의 자체 생명주기(TTL) 설정 방식에 대한 안내 코드/주석을 포함함.

## 4. 비기능 및 환경 요구사항 (Non-Functional Requirements)

* **배포 환경 최적화 (AntG 고려):** SvelteKit의 Full-stack 기능을 100% 활용하여 프론트엔드와 백엔드 API가 동일 오리진 내에서 통신하도록 밀결합 구조로 개발함.
* **모바일 UX 가드레일:** 모바일 브라우저 환경에서 긴 터치나 드래그 시 브라우저 자체 오버스크롤(바운스 효과)이나 텍스트 선택 이벤트가 발생하여 UI가 깨지지 않도록 CSS(`touch-action: pan-y`, `user-select: none`) 처리를 적용함.
* **보안:** 확장자 변조 방지 및 파일 사이즈 제한(예: 단일 이미지 최대 20MB) 로직을 서버사이드 검증 단에 포함함.

1. **소스 코드 세트:** 줄임표 없이 완성된 `+page.svelte`, `+server.js`, `cron.js` 등의 전체 파일 소스.
2. **패키지 명세:** `pdf-lib`, `node-cron` 등 필요한 모듈의 명확한 `npm install` 스크립트 가이드.
