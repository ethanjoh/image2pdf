# Role & Context
너는 SvelteKit(스벨트킷)과 Tailwind CSS 전문가이자, 풀스택 웹 개발자야. 
사용자가 이미지들을 업로드하거나 드래그앤드롭하여 하나의 PDF 파일로 병합 및 다운로드할 수 있는 웹 애플리케이션을 개발하려고 해. 이 코드는 'Ant Gravity(안티그래비티)' 환경에서 구동될 예정이므로, 백엔드와 프런트엔드가 SvelteKit 구조 안에서 완벽하게 상호작용해야 하며 모바일 브라우저(터치 입력 등)에서도 완벽히 작동해야 해.

다음 요구사항을 반영하여 전반적인 프로젝트 아키텍처와 핵심 파일(`+page.svelte`, `+page.server.js` 또는 `+server.js`)의 전체 코드를 작성해줘.

---

# 기술 스택 및 제약 조건
- **Framework:** SvelteKit (최신 안정 버전)
- **Styling:** Tailwind CSS (반응형 및 모바일 최적화)
- **PDF Library:** client-side 또는 server-side 중 이 작업에 가장 최적화되고 안정적인 라이브러리 사용 (예: jsPDF 또는 pdf-lib 등 사용 및 설치 안내 포함)
- **Storage:** 24시간 후 자동 삭제 메커니즘 필수 (Server-side 크론탭, 임시 폴더 주기적 삭제, 또는 Vercel Blob/Cloudflare R2 등 서버리스 환경을 고려한 TTL 설정 방식 제안)

# 핵심 기능 및 요구사항

### 1. 이미지 업로드 및 드래그앤드롭 (Drag & Drop)
- 데스크톱뿐만 아니라 **모바일 터치 환경에서도 정상 작동**하는 드래그앤드롭 영역 제공.
- 이미지 파일(PNG, JPG, JPEG)만 허용하며, 다중 선택 업로드 가능.
- 파일이 업로드되면 화면에 순서대로 **썸네일 리스트**를 그리드 형태로 표시.
- (선택 사항) 사용자가 리스트 내에서 순서를 마우스 드래그나 모바일 터치(Sortable 라이브러리 등 활용 가능)로 변경할 수 있도록 구조 설계.

### 2. PDF 변환 및 애니메이션 상태 관리
- 사용자가 'PDF 만들기' 버튼을 누르면 변환 작업이 시작됨.
- **작업 진행 중(Loading/Processing):** 사용자에게 현재 진행 상태를 알리는 세련된 CSS/SVG 기반 로딩 애니메이션(스피너 또는 프로그레스 바)을 노출하고, 버튼은 비활성화 처리.
- **작업 완료(Success):** 로딩 애니메이션이 사라지고 결과물을 다운로드할 수 있는 'PDF 다운로드' 버튼이 나타남.

### 3. 파일 보관 및 24시간 자동 삭제 (Storage & TTL)
- 업로드된 원본 이미지나 생성된 임시 파일은 **서버에 저장된 지 24시간이 지나면 자동으로 완전히 삭제**되어야 함.
- AntG 및 현대적인 배포 환경을 고려하여, Node.js의 `fs` 모듈을 이용한 서버 로컬 스토리지 기반 자동 스케줄러(예: `node-cron` 또는 앱 구동 시 체크) 구현 예시를 보여주거나, 클라우드 스토리지의 TTL 설정을 가이드해줘.

### 4. UI/UX 및 모바일 최적화
- 모바일(iOS, Android) 브라우저 화면에 꽉 차는 반응형 레이아웃.
- 터치 영역을 충분히 확보하고, 모바일 스크롤과 드래그 오퍼레이션이 충돌하지 않도록 처리.
- 스타일은 깔끔하고 모던한 UI 톤앤매너 유지.

# 출력 요구사항
1. 필요한 패키지 설치 명령어 (`npm install ...`)
2. 프로젝트 디렉토리 구조 (SvelteKit 기준)
3. 클라이언트 컴포넌트 코드 (`src/routes/+page.svelte`) - 상태 관리 및 UI, 애니메이션 포함
4. 서버 사이드 로직 및 API 코드 (`src/routes/api/pdf/+server.js` 또는 `+page.server.js`) - 이미지 처리, PDF 생성(서버에서 처리할 경우), 24시간 삭제 로직 포함
5. 각 코드 블록은 생략(줄임표) 없이 **전체 작동 가능한 코드**로 작성해줘.
